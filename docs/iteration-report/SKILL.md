# Skill: Iteration Report (6-Week)

## Purpose
Produce a structured document summarizing an entire iteration of work.

## Input
- All weekly summaries from the iteration (or ADR entries if summaries are missing)
- Current architecture state

## Output format
Produce a standalone markdown document:

---
# Iteration Report — [DATE RANGE]

## Executive Summary
[3-5 sentences on what the iteration accomplished architecturally]

## Decisions by Week
[Paste or summarize each weekly summary]

## Architecture Evolution
[What changed from start to end of iteration — services, contracts, infra]

## Recurring Themes
[Patterns that appeared across multiple weeks]

## Carried Forward
[Open questions or deferred decisions going into next iteration]

## Metrics (if available)
- ADRs logged: X
- Decisions reversed: X
- Services affected: X
---
